FoLiA ist ein auf XML-basiertes Format für die Annotation Linguistischer Dateien. FoLiA ist geeignet für die Speicherung und Austausch sölcher Ressourcen. Das Ziel ist um verschiedene Arten linguistischer Annotation zu unterstützen in einem einziges und umfangreiches Format basiert auf einem allgemeines Paradigma. FoLiA verpflichtet kein spezifischer Label-set, Sprache oder linguistische Theorie. Der Benutzer ist darin ganz frei, und das macht FoLiA sehr flexibel.

XML ist ein hierarchisches Format. FoLiA benützt diesen Struktur optimal und folgt wo möglich eine in-line hierarchische Methode. Wir erben das von unserem Vorgang, das D-Coi Format. D-coi behauptet auf einem Subset von TEI basiert zu sein. Wegen das umfangeiches Paradigma, FoLiA ist nicht kompatibel mit D-Coi. Es ist aber einfach FoLiA Dokumenten nach weniger expressive Formaten wie D-Coi oder unformatierten Text über zu setzen. Dafür bieten wir auch Konverter an.

Wichtige eigenschaften des FoLiA Formats sind:

  • Generalisiertes Paradigma - Wir benützen ein generalisiertes Paradigma und verwenden dabei so weinig möglich ad-hoc lösungen.
  • Expressivität - Das Format ist sehr expressif. Annotationen können sehr ausführlich ausgesetzt werden, wenn der Benützer möchte. Außerdem gibt es auch unterstützung um alternative Annotationen dar zu stellen und kann man Metadaten von Annotationen hinzufügen, wie zum Beispiel Informationen über dem Bearbeiter, das Zeitpunkt davon und das Vertrauen im Annotation.
  • Erweiterbar - Wegen das generalisiertes Paradigma und die Tatsache, das FoLiA kein Label-Sets verplichtet, ist das Format einfach erweiterbar.
  • Formalisiert - Das Format kan man validieren auf oberflächligem und tiefem Niveau. Das Format ist einfach von Rechner zu parsieren, es gibt dafür auch einige Bibliotheken und Hilfsprogramme.
  • Praktisch - Das FoLiA Format ist auf einem `bottom-up` Art entwickelt, zusammen mit Applikationen, HIlfsprogramme und Bibliotheken. Weil das Format sehr umfangreich ist, halten wir es so einfach wie möglich um damit die Annahme und Verwendung des Formats zu ermutigen.

FoLiA benützt in-line sowie stand-off Annotation. Inline Annotation gehört zu einzelne Wörter oder einzelne Strukturelemente, weil stand-off Annotation sich über mehrere Wörter umspannt. Stand-off annotation wird in getrennte Annonationslagen aufgenommen. FoLiA bietet so eine hohe flexibilität um verschiedene Arten linguistischer Annotation zu bedienen.

Zwei software Bibliotheken sind verfügbar um mit FoLiA dokument zu arbeiten in Ihren eigenen Sofware und scripts. Kontroliere oft ob eine neue Version verfügbar ist, weil sie noch voll in entwicklung sein.

  • pynlpl.formats.folia (für Python) - von Maarten van Gompel, distribuiert als Teil von PyNLPl.
    • Dokumentation der Python Bibliothek (auf Englisch)
    • Wir empfehlen pynlpl herunter zu laden vom Python Package Index oder github, auf diese Weise können Sie das einfach aktualisieren: $ sudo easy_install pynlpl
  • libfolia (für C++) - von Ko van der Sloot, Universität Tilburg
    • Noch kein Dokumentation verfügbar
    • Achtung: man benötigt libfolia um Frog sowie ucto zu kompilieren!

Für weitere unterstützung können Sie teilnehmen am FoLiA Mailing List.

FoLiA wird von verschiedene Projekten im Niederländischen und Flämischen Komputerlinguistikgesellschaft verwendet. Das großte Niederländische Korpus, SoNaR, ist im FoLiA Format veröffentlicht. Es gibt auch manche CLARIN Projekten, die FoLiA verwenden. FoLiA unterstützung ist integriert in mehrere Software-Pakete, wie ucto, Frog und Valkuil.net.

FoLiA ist von Maarten van Gompel entwickelt an der Universät Tilburg und heuzutage an der Radboud Universität Nijmegen, mit Hilfe von Antal van den Bosch, Ko van der Sloot, Martin Reynaert und anderen. Publikation von Forschung das FoLiA verwendet, kann FoLiA anführen als:

Maarten van Gompel (2012). FoLiA: Format for Linguistic Annotation. Documentation. ILK Technical Report 12-03. Available from http://ilk.uvt.nl/downloads/pub/papers/ilk.1203.pdf

FoLiA ist open-source en verfügbar unter der GNU Publike Lizenz v3.
